2. Trader Simulation Engine

Create a simulated trader engine that runs automatically on a timer.

Timer:
- Every 30–60 seconds, run one simulation cycle.

Each cycle, each trader independently:
- Reviews demo candles.
- Selects which timeframe/timeframes matter.
- Produces:
  - Bias: Bullish / Bearish / Neutral
  - Confidence: 0–100
  - Status: ANALYZING / THINKING / WAITING / RESEARCHING / DISCUSSING / IN TRADE
  - Current action
  - Internal reasoning
  - Recent decision
  - Open position decision: enter / hold / close / skip

3. Trader Strategy Logic

Use mock/rule-based strategy logic for now.

ICT Trader:
- Focus on liquidity sweeps, fair value gaps, order blocks, session timing.
- Should be patient and selective.
- Often skips trades if liquidity or confirmation is missing.

Trend Trader:
- Focus on higher-timeframe direction, pullbacks, momentum, continuation.
- Should prefer trading with trend.
- Should be moderately active.

Breakout Trader:
- Focus on ranges, consolidation, breakout, retest, volatility expansion.
- Should be more aggressive than the others.
- Should watch for compression and range breaks.

  - unrealized P/L
- Closed trade history:
  - entry
  - exit
  - direction
  - result
  - R multiple
  - balance change

Trading must be simulated only.

Rules:
- Traders can enter, hold, close, or skip.
- Do not allow unlimited trades.
- A trader should not open a second trade if already in a position.
- Use simple mock P/L calculations.
